# Claude Code CLI 入門ガイド

## ✨ Claude Code CLIとは？

**ターミナルから使える、コード作成・ファイル操作ができるAIアシスタント**

通常のChatGPTやClaude.aiと違い、あなたのローカルファイルを直接読み書きし、コマンドを実行できます。プログラミング作業を大幅に効率化する開発者向けツールです。

---

## 🎯 主な機能

### 1. ファイルの読み書き
```
「src/config.jsonを読んで設定を説明して」
「app.pyの関数名を一覧表示して」
```
→ ファイル内容を解析・説明

### 2. コードの生成・修正
```
「REST APIのエンドポイントを作成して」
「このバグを修正して」
「テストコードを追加して」
```
→ 要求に応じてコードを自動生成・編集

### 3. プロジェクト全体の検索・分析
```
「このプロジェクトで使われているAPIエンドポイントを全て探して」
「データベース接続している箇所をリストアップして」
```
→ 複数ファイルを横断して検索・分析

### 4. コマンド実行・自動化
```
「テストを実行して結果を報告して」
「依存関係をインストールして」
「ビルドしてエラーがあれば修正して」
```
→ シェルコマンドの実行と結果の解釈

## 📖 基本的な使い方

### 起動
```bash
claude
```

### 会話形式で指示
普通に日本語（または英語）で話しかけるだけ：
```
「現在のディレクトリ構造を教えて」
「package.jsonの依存関係を確認して」
```

### 確認と実行
Claude Codeは**必ず確認を取ってから実行**します：
```
Claude: 「以下の作業を行います：
1. config.jsonを読み込む
2. ポート番号を8080に変更
3. ファイルを保存
よろしいですか？(y/n)」

あなた: 「y」

Claude: 「修正しました。config.jsonのポート番号を8080に変更しました。」
```

## ⚠️ 使用上の重要なポイント

### ✅ やるべきこと

**1. 必ず確認してから承認**
- Claude Codeは勝手にファイルを変更しません
- 常に計画を提示して「y/n」で確認を求めます
- 不安な場合は「n」で中止できます

**2. 具体的に指示する**
```
❌ 悪い例: 「コードを直して」
✅ 良い例: 「main.pyの25行目のNoneチェックを追加して」
```

**3. 段階的に進める**
```
ステップ1: 「まず問題を調査して」
ステップ2: 「原因がわかったら修正案を提示して」
ステップ3: 「承認したら修正を実行して」
```

**4. バックアップを取る**
```bash
# 重要なファイルは事前にバックアップ
cp database.json database.json.backup
```

### ❌ 避けるべきこと

**1. 本番環境での直接実行**
- まずは開発環境やテスト環境で試す
- 動作確認してから本番に適用

**2. 機密情報の直接記載**
- パスワードやAPIキーは環境変数で管理
- `.env`ファイルは`.gitignore`に追加

**3. 盲目的な信頼**
- AIの提案も完璧ではありません
- 重要な変更は内容を確認してから承認
